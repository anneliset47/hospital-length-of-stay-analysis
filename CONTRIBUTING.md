# Contributing

Thank you for your interest in improving this project.

## Ways to Contribute

- Improve statistical rigor or model diagnostics
- Add reproducibility enhancements
- Improve documentation clarity and project communication
- Propose additional healthcare operations features

## Development Guidelines

1. Keep changes focused and well-scoped.
2. Preserve existing variable names and column references unless necessary.
3. Update README/docs when behavior or structure changes.
4. Use clear commit messages.

## Pull Request Checklist

- [ ] Changes are minimal and targeted
- [ ] Notebook still runs end-to-end
- [ ] Documentation is updated (if relevant)
- [ ] No sensitive data added

## Reporting Issues

Please open an issue with:

- Expected behavior
- Actual behavior
- Reproduction steps
- Environment details (OS, R version, Jupyter/VS Code)
